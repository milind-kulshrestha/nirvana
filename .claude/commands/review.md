# Code Review with Project Context

Perform code review considering project standards:

1. **Check Against Architecture Documentation**
   - **Backend:** Follows patterns in docs/reference/architecture/backend.md?
   - **Frontend:** Consistent with docs/reference/architecture/frontend.md?
   - **Database:** Matches schema in docs/reference/architecture/database.md?
   - **External APIs:** Proper usage per docs/reference/external/

2. **Quality Checks**
   - Code clarity and maintainability
   - Error handling and edge cases
   - Performance considerations
   - Security best practices

3. **Documentation**
   - Adequate inline comments?
   - Docstrings complete?
   - Reference docs updated if needed?
   - Contributing guidelines followed?

4. **Testing**
   - Tests included and comprehensive?
   - Coverage adequate?
   - Test quality and maintainability?

5. **Project Standards**
   - Check docs/contributing/ for style guidelines
   - Verify development workflow compliance

Provide specific, actionable feedback with examples and reference relevant documentation.
