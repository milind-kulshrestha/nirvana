# Update Changelog

After completing any work, automatically update docs/project/changelog.md:

1. Add entry with format:
```
   ## [Date] - [Session/Feature Name]
   ### Added
   - New features or files
   
   ### Changed
   - Modified functionality
   
   ### Fixed
   - Bug fixes
   
   ### Technical Details
   - Implementation notes
   - Performance impacts
   - Dependencies added/removed
```

2. Keep entries reverse chronological (newest first)
3. Use clear, specific descriptions
4. Link to relevant files or commits
