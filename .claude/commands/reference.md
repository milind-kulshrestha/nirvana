# Update Reference Documentation

Maintain reference documentation in docs/reference/ following the organized structure:

## Architecture Documentation (docs/reference/architecture/)
- **backend.md** - FastAPI structure, routes, models, OpenBB integration
- **frontend.md** - React structure, state management, routing, components  
- **database.md** - PostgreSQL schema, relationships, migrations

## External API Documentation (docs/reference/external/)
- **openbb.md** - OpenBB Python library API reference, examples, technical indicators

## When to Update
- Adding new modules or major components
- Refactoring system architecture
- Changing development patterns or conventions
- Adding new external API integrations
- Modifying database schema

## Update Process
1. Identify which reference docs are affected
2. Update technical details and code examples
3. Ensure cross-references between docs are accurate
4. Update docs/README.md navigation if new sections added
