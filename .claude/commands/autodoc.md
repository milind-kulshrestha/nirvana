# Auto-Documentation Protocol

Run automatically after completing work sessions:

1. **Update Project Documentation**
   - **docs/project/changelog.md** - Add session entry with changes
   - **docs/project/status.md** - Mark completed tasks, update progress

2. **Update Reference Documentation (if needed)**
   - **docs/reference/architecture/** - New modules, architecture changes
   - **docs/reference/external/** - New API integrations or changes
   - **docs/getting-started/** - Setup or workflow changes

3. **Update Guides (if applicable)**
   - **docs/guides/authentication.md** - Auth flow changes
   - **docs/guides/database-setup.md** - Schema or migration changes

4. **Create Session Summary**
   - What was accomplished
   - What's next
   - Any decisions made or trade-offs
   - Documentation files updated

Ask: "Should I commit these documentation updates?"
